# 小程序日报系统前端实现

## 概述
本 PR 实现了林麝健康监测小程序的日报系统前端，包括日报列表、动物详情、诊疗记录、饲养记录展示以及添加记录功能。

## 实现的功能

### 1. 日报列表页面 (`pages/daily-report/`)
- **日期选择器**: 支持左右滑动查看最近7天的日报数据
- **统计概览**: 显示存栏总数、健康/需关注/异常数量、平均活动评分
- **今日摘要**: 展示重要提醒和亮点
- **动物列表**: 显示圈舍内所有动物的基础信息和数据看板
- **悬浮按钮**: 快速添加记录

### 2. 动物详情页面 (`pages/animal-detail/`)
- **基础信息栏**: 显示耳标号、性别、年龄、健康状态（绿黄红标识）
- **每日数据看板**: 
  - 活动评分及等级
  - 吃料剩余百分比
  - 喝水量统计
  - 进食次数
- **活动趋势图表**: 使用 echarts 展示周/月活动趋势
- **标签页切换**: 
  - 概览: 诊疗和饲养记录摘要
  - 诊疗记录: 诊断、用药列表、剩余天数
  - 饲养记录: 今日已完成(✓)、今日待办、明日计划

### 3. 添加记录页面 (`pages/add-record/`)
- **记录类型**: 观察记录、饲养记录、诊疗记录
- **多媒体支持**:
  - 文本输入（最多500字）
  - 图片上传（最多9张）
  - 语音录制（最长60秒）
- **动物选择**: 支持多选（从圈舍进入时）

### 4. 公共服务和工具
- **API 服务** (`services/dailyReport.js`): 封装所有日报相关接口
- **工具函数** (`utils/util.js`): 日期格式化、年龄计算、健康状态颜色等

## 技术栈
- 微信小程序原生开发
- echarts-for-weixin 图表组件
- 后端 API: `/api/v2/mp/*`

## 页面路由
```
/pages/daily-report/daily-report    - 日报列表
/pages/animal-detail/animal-detail  - 动物详情
/pages/add-record/add-record        - 添加记录
```

## API 接口
- `GET /mp/daily-reports` - 获取日报列表
- `GET /mp/daily-reports/:id` - 获取日报详情
- `GET /mp/animals/:id` - 获取动物信息
- `GET /mp/animals/:id/dashboard` - 获取数据看板
- `GET /mp/animals/:id/medical-records` - 获取诊疗记录
- `GET /mp/animals/:id/feeding-records` - 获取饲养记录
- `POST /mp/animals/:id/records` - 添加记录
- `POST /mp/upload/image` - 上传图片
- `POST /mp/upload/voice` - 上传语音
- `GET /mp/sheds/:id/animals` - 获取圈舍动物列表

## UI 设计
遵循客户提供的 UI 设计规范：
- 基础信息栏：耳标号、性别年龄、健康状态（绿黄红）
- 每日数据看板：活动评分、吃料剩余%、喝水量
- 治病用药：诊断、药物列表、剩余天数
- 饲养记录：今日重点（✓）、明天要做（☐）
