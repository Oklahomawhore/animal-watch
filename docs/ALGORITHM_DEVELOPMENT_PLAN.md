# 林麝健康检测 - 算法开发计划

**项目**: animal-watch / lin-she-health-monitor  
**分支**: feature/algorithm-development  
**创建时间**: 2026-03-12  
**目标摄像头**: GF6830765_5, GF6830765_10 (2区母麝圈)

---

## 一、算法模块概述

### 1. 运动量检测
**目标**: 检测林麝的运动量，判断活动状态  
**技术方案**: 目标检测 + 方框位移追踪

### 2. 进食检测
**目标**: 检测林麝是否在进食，评估进食量  
**技术方案**: 食槽检测 + 图像分割 + 像素变化检测

### 3. 饮水检测
**目标**: 检测林麝的饮水行为和饮水量  
**技术方案**: 水盆检测 + 目标检测 + 姿态识别

## 二、数据标注计划

### 2.1 标注数据类型

#### 模块1: 运动量检测
| 数据类型 | 标注内容 | 数量需求 |
|---------|---------|---------|
| 目标检测框 | 林麝边界框 (bbox) | 500-1000张 |
| 追踪ID | 同一林麝的连续帧ID | 视频序列 |
| 关键点 | 林麝姿态关键点 (可选) | 200张 |

#### 模块2: 进食检测
| 数据类型 | 标注内容 | 数量需求 |
|---------|---------|---------|
| 目标检测框 | 林麝边界框 | 500-1000张 |
| 食槽分割 | 食槽区域多边形分割 | 300-500张 |
| 食物分割 | 食物区域像素级分割 | 300-500张 |
| 行为标签 | 进食/未进食/离开 | 1000张 |

#### 模块3: 饮水检测
| 数据类型 | 标注内容 | 数量需求 |
|---------|---------|---------|
| 目标检测框 | 林麝边界框 | 500-1000张 |
| 水盆分割 | 水盆区域多边形分割 | 300-500张 |
| 姿态标签 | 饮水姿态/站立/离开 | 500张 |

### 2.2 标注方法

#### 工具选择
- **LabelImg**: 目标检测框标注 (YOLO格式)
- **LabelMe**: 多边形分割标注
- **CVAT**: 视频追踪标注 (可选)

#### 标注格式

**YOLO目标检测格式**:
```
<class_id> <x_center> <y_center> <width> <height>
0 0.5 0.5 0.3 0.4
```

**COCO分割格式**:
```json
{
  "annotations": [{
    "id": 1,
    "image_id": 1,
    "category_id": 1,
    "bbox": [x, y, width, height],
    "segmentation": [[x1,y1,x2,y2,...]],
    "area": 1234,
    "iscrowd": 0
  }]
}
```

#### 标注流程
1. 从 GF6830765_5 和 GF6830765_10 通道下载图片
2. 使用 LabelImg 标注林麝边界框
3. 使用 LabelMe 标注食槽/水盆分割区域
4. 导出 YOLO 和 COCO 格式
5. 数据增强 (旋转、翻转、亮度调整)

## 三、训练框架选择

### 3.1 目标检测框架
| 框架 | 模型 | 优点 | 适用场景 |
|------|------|------|---------|
| **YOLOv8** | yolov8n/s/m | 速度快，部署简单 | 实时检测首选 |
| **MMDetection** | Faster R-CNN | 精度高，可定制 | 高精度需求 |
| **Detectron2** | Mask R-CNN | Facebook官方，分割强 | 分割任务 |

**推荐**: YOLOv8 (兼顾速度和精度)

### 3.2 图像分割框架
| 框架 | 模型 | 优点 |
|------|------|------|
| **YOLOv8-seg** | yolov8n-seg | 速度快，端到端 |
| **MMDetection** | Mask R-CNN | 精度高 |
| **Segment Anything** | SAM | 零样本分割 |

**推荐**: YOLOv8-seg (与检测统一框架)

### 3.3 技术栈
```

Ultralytics (YOLOv8)
OpenCV
NumPy / Pillow
Albumentations (数据增强)

```

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 标注数据不足 | 模型精度低 | 数据增强 + 主动学习 |
| 遮挡严重 | 检测困难 | 多摄像头融合 + 追踪算法 |
| 光线变化 | 误检率高 | 图像预处理 + 自适应阈值 |
| 实时性不足 | 系统卡顿 | 模型量化 + 边缘计算 |
